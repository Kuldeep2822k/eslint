## 2025-03-05 - [CRITICAL] HTML Injection/XSS in HTML Formatter
**Vulnerability:** The HTML formatter (`lib/cli-engine/formatters/html.js`) had a Cross-Site Scripting (XSS) vulnerability. While `message`, `filePath`, and `summary` strings from linting results were properly HTML-escaped using `encodeHTML()`, `ruleId` and `ruleUrl` were embedded directly into the HTML string unescaped. If a user ran ESLint on code that triggered a malicious custom rule designed to output a crafted string for `ruleId` or `ruleUrl`, it could execute arbitrary code when the resulting HTML report was viewed in a browser.
**Learning:** Even when basic templating handles some variables defensively, variables like IDs and URLs that often seem "safe" or "fixed" must be escaped if their content can be derived from externally-supplied data (such as third-party lint rules).
**Prevention:** Always apply the same HTML-escaping (e.g., `encodeHTML`) to all dynamic variables interpolated into HTML strings, even for fields like `ruleId` that are typically constrained formats.
