## 2024-05-24 - [Cross-Site Scripting (XSS) in HTML Formatter]
**Vulnerability:** The HTML formatter (`lib/cli-engine/formatters/html.js`) interpolated `ruleId` and `ruleUrl` directly into the generated HTML output without escaping them. Since these values can be defined by custom rules, a maliciously crafted rule metadata could inject arbitrary HTML/JavaScript into the generated report.
**Learning:** Even internal metadata from plugins (like rule IDs and URLs) must be treated as untrusted user input when generating HTML reports. The formatter already correctly encoded the `message` field, but missed the rule-specific attributes.
**Prevention:** Always use the `encodeHTML` utility for *all* dynamically interpolated variables within the HTML reporter template, regardless of whether the source seems "safe" (like rule metadata). When testing HTML entity encoding, be aware that libraries like Cheerio will automatically decode entities and parse attribute values. Tests must verify the raw string output (e.g., using `result.includes()`) to confirm the entities are actually present.
