## 2024-05-18 - [XSS in CLI HTML Formatter]
**Vulnerability:** Found an XSS vulnerability in `lib/cli-engine/formatters/html.js` where `ruleId` and `ruleUrl` were not properly escaped before being injected into the HTML report template. This could allow execution of arbitrary JavaScript if an attacker crafts malicious rule IDs or URLs.
**Learning:** Even though `message`, `filePath`, and `summary` were correctly escaped using `encodeHTML`, `ruleId` and `ruleUrl` were overlooked. It is critical to ensure that *all* externally derived inputs are properly escaped when generating HTML output in CLI formatters to avoid XSS vulnerabilities, particularly those that come from dependencies or configurations outside the formatter itself.
**Prevention:** In CLI formatters generating HTML, ensure that a unified HTML encoding function is applied to every variable injected into the template. When testing HTML generation using libraries like Cheerio, remember that it automatically parses entities, so raw string assertions (e.g., `assert.include`) should be used to verify exact string-level encoding of attributes and content.
